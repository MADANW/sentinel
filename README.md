# algo-bot

Hybrid AI trading system. You make the directional call; the bot handles execution, sizing, and risk.

## Stack

- **Claude API** → morning bias (bullish/bearish/neutral + confidence)
- **Python backend** → EMA crossover detection + Alpaca bracket orders
- **Next.js dashboard** → trade journal + Supabase
- **Hardcoded risk engine** → 1% risk/trade, 2% daily loss kill switch, 3 trades/day max

## Quick Start

```bash
# 1. Clone and set up secrets
cp .env.example .env
# Fill in .env — never commit it

# 2. Install Python dependencies
cd backend
pip install -r requirements.txt

# 3. Install pre-commit hooks (runs secret scanning before every commit)
pip install pre-commit
pre-commit install

# 4. Run tests
pytest

# 5. Start the bot (paper trading by default)
python main.py
```

## Security

See [SECURITY.md](docs/SECURITY.md) for the full threat model, secret rotation runbook, and VPS hardening checklist.

**TL;DR:**
- Secrets live in `.env` only — never in code or git history
- `TRADING_ENV=paper` by default — must explicitly set `=live` for real money
- Risk limits are hardcoded and cannot be overridden at runtime
- Pre-commit hooks and CI scan every commit for secrets

## Risk Limits (hardcoded, not configurable)

| Limit | Value |
|-------|-------|
| Risk per trade | 1% of equity |
| Daily loss kill switch | 2% of equity |
| Max trades per day | 3 |
| Min Claude confidence to trade | 60% |

## Getting Started

### Prerequisites

- Python 3.12
- Node.js 20+
- A [Supabase](https://supabase.com) project (free tier works)
- [Alpaca](https://alpaca.markets) paper trading account (free)
- [Anthropic](https://console.anthropic.com) API key

### Setup

```bash
# 1. Clone
git clone https://github.com/<you>/algo-bot && cd algo-bot

# 2. Python environment
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate

# 3. Install Python dependencies (hash-verified)
pip install --require-hashes -r backend/requirements.txt

# 4. Install pre-commit hooks (runs secret scanning before every commit)
pip install pre-commit && pre-commit install

# 5. Environment variables
cp .env.example .env
# Edit .env — fill in:
#   ALPACA_API_KEY, ALPACA_SECRET_KEY  (use paper keys)
#   ANTHROPIC_API_KEY
#   SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY

# 6. Run Supabase migrations
# Apply these in the Supabase dashboard SQL editor (or via Supabase CLI):
#   supabase/migrations/001_create_trades.sql
#   supabase/migrations/002_create_pipeline_runs.sql

# 7. Train the model (required before first run)
# Fetches ~3 years of OHLCV for each ticker and trains an XGBoost classifier
# on 12 features (momentum/volume + Hurst/OU/regime). Per-ticker chronological
# split then concat — no cross-ticker lookahead.
# Generates models/classifier.pkl (local only) and models/classifier.sha256 (commit this).
# Free-tier Alpaca accounts should set ALPACA_DATA_FEED=iex (the default).
python -m backend.scripts.train_model --tickers SPY,QQQ,IWM --days 756

# 8. Run tests
pytest backend/tests/

# 9. Paper trade (default — safe, uses Alpaca paper account)
TRADING_ENV=paper python main.py --ticker SPY

# 10. Dashboard
cd dashboard
cp .env.local.example .env.local
# Edit .env.local — fill in SUPABASE_URL and SUPABASE_SERVICE_ROLE_KEY
npm ci
npm run dev
# Open http://localhost:3000
```

### Why classifier.pkl is not in the repo

`models/classifier.pkl` is a Python pickle file. Pickle files can **execute arbitrary code** when loaded — distributing one in a public repository would be a critical security vulnerability. Every engineer trains their own model locally using `train_model.py`.

The committed `models/classifier.sha256` file is the SHA-256 hash of the pkl generated by your local training run. The runtime (`model.py`) verifies this hash against the pkl bytes **before** calling `pickle.load()`. A hash mismatch immediately raises `ModelTamperingError` and exits — no model is loaded.

**Never accept `classifier.pkl` from an external source.** See `docs/BRAIN.md → Pickle Security Note`.
